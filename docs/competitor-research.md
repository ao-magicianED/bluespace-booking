## 1. 予約フロー分析

### 1.1 リクエスト承認方式 vs 即時予約

| プラットフォーム | 即時予約 | リクエスト承認制 | ホスト選択可能性 |
|---|---|---|---|
| **よやクル** | ✓ | ✓ | ✓（選択可） |
| **スペースマーケット** | ✓ | ✓（デフォルト当日予約は可） | ✓（選択可） |
| **インスタベース** | ✓（今すぐ予約） | ✓（予約リクエスト制） | ✓（選択可） |
| **UPNOW** | ✓ | ✓（リクエスト状態で保留） | ✓（カスタマイズ可能） |

**結論**: 全プラットフォームが両方に対応。ホスト側で選択可能が標準。

### 1.2 時間単位の設定方式

| プラットフォーム | 最小単位 | 設定可能粒度 | 最低利用時間設定 |
|---|---|---|---|
| **よやクル** | 1時間単位推奨 | 柔軟（テンプレート複数） | ✓（カスタマイズ可） |
| **スペースマーケット** | 1時間単位 | 当日は30分後以降を時間単位 | ✓（プラン毎に設定） |
| **インスタベース** | **30分単位** | 0.5h（30分）から24hまで30分単位 | ✓（0.5h〜設定可） |
| **UPNOW** | 時間単位 / 全日貸 | カスタマイズ可能 | ✓（カスタマイズ可） |

**結論**: インスタベースが最も細かい（30分単位）。スペースマーケットは当日予約で30分後以降対応。

## 2. 空き状況表示UI

| プラットフォーム | カレンダー形式 | UI/UX特徴 |
|---|---|---|
| **よやクル** | ✓（月カレンダー） | 管理画面はカレンダーで貸出不可設定可能 |
| **スペースマーケット** | ✓（月カレンダー + 日カレンダー） | ゲスト向け：ひと目で空き状況判定可能。ホスト向け：ダブルブッキング完全防止機能 |
| **インスタベース** | ✓（空室カレンダー） | 日付・時間帯を選択すると自動料金計算（最安プラン自動選択） |
| **UPNOW** | ✓（Google Calendar iframe） | Google Calendar統合で複数チャネル一元管理可視化 |

**結論**: 全社カレンダー形式が標準。スペースマーケットの「日カレンダー」分離が詳細表示で優れている。

## 3. 決済方法 & 決済タイミング

| プラットフォーム | 決済方法 | 決済タイミング | 手数料ホスト負担 |
|---|---|---|---|
| **よやクル** | クレジットカード、PayPay | 予約確定時 | ✗（0円） |
| **スペースマーケット** | クレジットカード（ゲスト決済） | 予約確定時 | ✓（30%） |
| **インスタベース** | クレジットカード + 支払手数料 | 予約確定時 | ✓（35% 成約手数料） |
| **UPNOW** | クレジットカード | 予約確定時 | ✓（決済手数料3.6% + サイト手数料3% = 計6.6%） |

**結論**: 決済タイミングは「予約確定時」で統一。よやクルが唯一ホスト0円手数料（月額制）。

## 4. キャンセルポリシーの標準パターン

| プラットフォーム | パターン数 | 標準オプション |
|---|---|---|
| **よやクル** | カスタマイズ可 | （公開情報なし） |
| **スペースマーケット** | 4種 | 易しい / 普通 / 厳しい / カスタム |
| **インスタベース** | カスタマイズ可 | 例：7日前50% / 前日100% / 当日100% |
| **UPNOW** | 3パターン + カスタム | プリセット3 + 完全カスタム可能 |

**標準的な階段型キャンセル料（業界相場）**:
- 7日以上前: 0%（無料キャンセル）
- 3〜7日前: 50%
- 1〜3日前: 75%
- 前日〜当日: 100%

**結論**: スペースマーケットの「4つのプリセット」が選択式で UX最適。カスタム必須なら手数で柔軟な UPNOW がやや優位。

## 5. 手数料率（運営者側負担）

| プラットフォーム | 手数料体系 | 手数料率 | 振込手数料 |
|---|---|---|---|
| **よやクル** | 月額定額制 | **¥4,980/月（税込）** | ホスト負担ゼロ |
| **スペースマーケット** | 成約時手数料 | **30%**（税抜売上） | スペースマーケット負担 |
| **インスタベース** | 成約時手数料 | **35%**（売上） | ホスト実費精算 |
| **UPNOW** | 売上連動手数料 | **6.6%**（決済手数料3.6% + サイト手数料3%） | ホスト実費精算 |

### 収益試算例（月10万円売上）

```
よやクル:  ¥100,000 - ¥4,980 = ¥95,020（月額固定）
スペースマーケット: ¥100,000 × (1 - 0.30) = ¥70,000
インスタベース: ¥100,000 × (1 - 0.35) = ¥65,000
UPNOW: ¥100,000 × (1 - 0.066) = ¥93,400
```

**結論**: UPNOW（6.6%）が最安。次によやクル（月額4,980円）。スペースマーケット（30%）は利用規模に応じて変動。

## 6. 価格設定の仕組み

| プラットフォーム | 曜日別価格 | 時間帯別価格 | ダイナミックプライシング | 特別営業 |
|---|---|---|---|---|
| **よやクル** | ✓ | ✓（テンプレート） | × | ✓（カスタム） |
| **スペースマーケット** | ✓ | ✓ | × | ✓（特別営業機能） |
| **インスタベース** | ✓ | ✓ | ✗（自動選択） | ✓（複数プラン組み合わせ） |
| **UPNOW** | ✓ | ✓（時間帯別料金） | × | ✓（早割・直前割） |

### プラン運用の特徴

- **よやクル**: テンプレート提供で曜日・時間帯の定義が楽
- **スペースマーケット**: 「特別営業」で繁忙期・閑散期の値上げ/値下げが直感的
- **インスタベース**: 複数プランを組み合わせ、ゲスト側で最安プラン自動選択（透明性高）
- **UPNOW**: 早割・直前割の設定が簡潔

**結論**: 全プラットフォームが曜日・時間帯別価格に対応。AI/データ駆動のダイナミックプライシング機能は実装なし（手動設定）。

## 7. Googleカレンダー連携

| プラットフォーム | カレンダー連携 | 方式 | 同期速度 | 双方向性 |
|---|---|---|---|---|
| **よやクル** | ✓ | Google Calendar統合（API） | リアルタイム | ✓ |
| **スペースマーケット** | ✓ | Google Calendar / iCal（iCalendar形式） | 数時間遅延 | ▲（iCalは一方向） |
| **インスタベース** | ⚠️ | （公開情報限定） | （未確認） | （未確認） |
| **UPNOW** | ✓ | Google Calendar連携 | リアルタイム | ✓ |

### スペースマーケットの連携詳細

- **インポート**: Google Calendar の外部カレンダーをスペースマーケット上で表示（二重予約防止）
- **エクスポート**: スペースマーケットの予約を Google Calendar に書き出し
- **iCal対応**: Mac Calendar、Outlook、Airbnb、Booking.com 等と連携可能
- ⚠️ **注意**: iCal形式では「スペースマーケットの不可設定」がエクスポートされない、遅延あり

**結論**: よやクルと UPNOW が最もシンプル。スペースマーケットは複数形式対応で柔軟だが設定複雑。

## 8. 業界標準パターン総括

### 予約フロー
- **デフォルト**: リクエスト承認制（安全側）+ 即時予約オプション
- **当日対応**: 当日は「現在時刻+30分以降」から受け付け

### UI/UX
- **空き状況**: 月カレンダー（概要）+ 日カレンダー（詳細時間帯）の2段階表示
- **決済**: 予約確定時に自動決済（PayPay / クレジットカード併用）

### 手数料構造
- **プラットフォーム型**: 30～35% の成約手数料（Airbnb / Booking.com も同水準）
- **SaaS型**: 月額定額 or 低手数料（5～10%）で運営者に有利

### キャンセルポリシー
- **推奨**: 3～4段階の選択式プリセット + カスタム可能
- **理由**: ホスト都度判断コスト削減 + ゲスト予測可能性向上

### 価格設定
- **標準機能**: 曜日・時間帯別 + 特別営業（繁忙期対応）
- **ダイナミックプライシング**: 実装例なし（需要予測型はまだ成熟度低い）

## 9. ブルーステージ自社システムで「真似すべきUXパターン」トップ5

### ✅ 推奨実装パターン（優先度順）

#### **1. 「3つの手数料層」設計でブルーステージ事業拡大に対応**
```
現在: Stripe決済手数料 3.6% + サイト0% = 計3.6%（業界最安）
⚡ 強み活用: 
  - 「手数料ゼロ」を大前提に、長期定期利用者向けの月額割引プラン追加
  - 例: 月10万利用で月額¥3,000 固定化 = UPNOW と同等競争力
```

#### **2. スペースマーケット型「4つのキャンセルポリシープリセット」**
```
ブルーステージの7拠点で「易しい／普通／厳しい／カスタム」を
各拠点の繁閑パターンで選択式にする。
理由: ホスト負担削減 + ゲスト安心度向上
```

#### **3. インスタベース型「日カレンダー + 時間帯別自動料金計算」**
```
ゲストが「利用日」→「時間帯」を選ぶと、複数プランから
自動で最安料金を表示。
実装: 前提として「最低利用時間」「複数日割」プラン複合システム必須。
```

#### **4. スペースマーケット / UPNOW 式「Google Calendar 双方向連携」**
```
現在: Googleカレンダーから入力 → Stripe 決済 → 自社予約システム確定
⚡ 改善案:
  - Google Calendar iframe を詳細ページに埋め込み（スペースマーケット参考）
  - iCal フィード公開で、ゲスト用 Googleカレンダー共有対応

理由: Airbnb / Booking.com との並行運用ユーザーの UX統一
```

#### **5. よやクル型「複数チャネル一元管理 + LINE通知」**
```
ブルーステージ事業規模（7拠点）で複数の外部プラットフォーム併用前提の時、
「自社予約システム = HUB」化する。

実装:
  - LINE Official Account で予約確定・キャンセル・リマインド配信
  - Discord bot（既存）で新着予約通知 + Gemini判定連携
  - Google Apps Script で予約データを Notion 管理ハブに集約
```

## 補足: ブルーステージの自社システム（Stripe + Google Calendar）の利点

1. **手数料が業界最安**: Stripe 3.6% は UPNOW（6.6%）の半分以下
2. **外部プラットフォーム依存なし**: 長期利益率が格段に優位
3. **API統合の自由度**: Google Calendar / Notion / Discord との連携が CLAUDE.md ですでに実装済み
4. **スケーラビリティ**: 拠点数増加時も増分コスト最小

**推奨戦略**: UPNOW の「6.6% 構造」を研究しつつ、ブルーステージの手数料優位性（3.6%）を「長期定期利用割」で顧客に還元し、プラットフォーム間の「流出防止」を図る。
