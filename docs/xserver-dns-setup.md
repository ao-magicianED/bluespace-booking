# Xサーバーで bluespacerental.com を予約システムに接続

対象ドメイン: **bluespacerental.com**（Xサーバー管理・現在パーク状態）
接続先: Vercel（予約システム本体）
所要: 5分 + 反映待ち（数分〜1時間）

## Xサーバーでの操作場所

1. https://www.xserver.ne.jp/login_server.php から **サーバーパネル** にログイン
2. 右上「ドメイン」→「**DNSレコード設定**」
3. `bluespacerental.com` を選択 →「**DNSレコード追加**」タブ

## 追加するレコード（2件）

### ① ルートドメイン用

| 項目 | 値 |
|---|---|
| ホスト名 | （**空欄**のまま） |
| 種別 | **A** |
| 内容 | `76.76.21.21` |
| 優先度 | 0（そのまま） |

### ② www用

| 項目 | 値 |
|---|---|
| ホスト名 | `www` |
| 種別 | **A** |
| 内容 | `76.76.21.21` |
| 優先度 | 0（そのまま） |

> `76.76.21.21` はVercelの受け口（世界中に分散した固定IP）です。両方とも同じIPでOK。

## 注意

- 既存のMXレコード（メール用）はそのままで構いません（このドメインでメールは使っていないため）。
- もし「同じホスト名のAレコードが既にある」と出たら、それを上記の内容に**修正**してください。
- 反映後、VercelがSSL証明書（https化）を自動で発行します。何もしなくてOK。

## 完了記録（2026-06-11 実施済み）

- ✅ Xサーバー DNSレコード設定で既存Aレコードを書き換え:
  - `bluespacerental.com` A → 76.76.21.21（Vercel）
  - `www.bluespacerental.com` A → 76.76.21.21（Vercel）
  - `*`（ワイルドカード）はXサーバーIPのまま（未使用のため）
- ✅ 権威サーバー（ns1.xserver.jp）で76.76.21.21を返すことを確認
- ✅ Vercelが自動SSL発行 → https://bluespacerental.com / https://www.bluespacerental.com とも 200 OK
- ✅ `NEXT_PUBLIC_SITE_URL` を `https://bluespacerental.com` に変更し本番デプロイ済み
- 予約システム本番URL: **https://bluespacerental.com**

## 今後（2027年末・コスト削減）

Xサーバー契約は2028年1月まで。それまでにこのドメインをCloudflare Registrarへ移管（Xサーバーサポートに特典解除依頼→1年分更新料約¥1,500→移管）すれば、Xサーバー解約しても年¥1,600だけで運用継続できる。

## Xサーバー APIキーについて

**このドメイン接続にAPIキーは不要です。** 上記のDNS画面操作だけで完結します。
APIキーはXサーバーのサーバー/DNSを操作するための鍵で、予約システム本体（Vercel上で動く）とは無関係なので、`.env` には入れません。あおさんの手元で保管しておいてください。
